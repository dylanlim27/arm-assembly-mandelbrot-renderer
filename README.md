# ARM Assembly Mandelbrot Renderer

An optimized, bare-metal ARM assembly program that renders an ASCII representation of the **Mandelbrot Set** using fixed-point arithmetic. 

This project was developed for the **VisUAL ARM Emulator**, demonstrating low-level optimization, register allocation strategies, and fractional math emulation on an integer-only processor.

## 🚀 Key Features

*   **Low-Level Portability**: Written in pure ARM assembly compatible with standard v4T+ architectures (specifically tested on the VisUAL emulator).
*   **Integer-Based Fixed-Point Math**: Emulates floating-point operations using **8.8 fixed-point arithmetic** (scaling real numbers by $256$).
*   **Optimized Assembly Routines**: Loops are structured using test-at-bottom conditionals to minimize branching overhead.
*   **Configurable Viewport**: Boundaries (`xstart`, `xend`, `ystart`, `yend`) and pixel resolution density (`xstep`, `ystep`) are easily configurable in the data section to render different levels of zoom.

## 📐 Technical Architecture & Implementation

### 1. The Mandelbrot Algorithm
The Mandelbrot set consists of complex numbers $c = x + iy$ for which the function $z_{n+1} = z_n^2 + c$ does not diverge when iterated from $z_0 = 0$.
Representing $z$ as $r + ii$ (real and imaginary parts):
*   $r_{next} = r^2 - i^2 + x$
*   $i_{next} = 2ri + y$

### 2. Fixed-Point Arithmetic (Scaling by 256)
Since standard ARM CPUs lack hardware floating-point support, this project uses **8.8 fixed-point representation** (where the lower 8 bits represent the fractional part, and the upper 24 bits represent the integer part). 

To prevent numeric overflow when multiplying two scaled numbers, the product is divided by $256$ (implemented via a high-performance **Arithmetic Shift Right by 8 bits**, `ASR #8`):
*   **Real Square**: $r^2 = (r \times r) \gg 8$
*   **Imaginary Square**: $i^2 = (i \times i) \gg 8$
*   **Cross Product**: $2ri = ((r \ll 1) \times i) \gg 8$

### 3. Register Allocation Strategy
To maximize performance and avoid memory bottlenecking, all loop variables and mathematical values are stored directly in CPU registers:

| Register | Variable Name | Purpose |
| :--- | :--- | :--- |
| **`R0`** | `tmp` / `SWI_Arg` | Temporary calculations, arithmetic results, and syscall parameters |
| **`R1`** | `x` | Current X coordinate on the complex plane |
| **`R2`** | `xend` | Limit boundary for the X loop |
| **`R3`** | `xstep` | Column increment size |
| **`R4`** | `y` | Current Y coordinate on the complex plane |
| **`R5`** | `yend` | Limit boundary for the Y loop |
| **`R6`** | `ystep` | Row increment size |
| **`R7`** | `it` | Iteration escape counter (initialized to `2048`) |
| **`R8`** | `r` | Real part of $z_n$ |
| **`R9`** | `i` | Imaginary part of $z_n$ |
| **`R10`** | `r2` | Square of the real part ($r^2$) |
| **`R11`** | `i2` | Square of the imaginary part ($i^2$) |

## 🛠️ How to Run & Emulate

1.  Download and install the **VisUAL ARM Emulator** (or any emulator supporting standard UAL/ARM assembly).
2.  Open [mandelbrot.s](file:///c:/Users/dylan/OneDrive/Coding%20Projects/arm-assembly-mandelbrot/mandelbrot.s) in the emulator.
3.  Ensure the emulator settings match standard ARM instructions.
4.  Execute/Run the code.
5.  View the ASCII art rendered directly in the emulator's **custom console/log output** (via VisUAL's `SWI 3` printing calls).

### VisUAL Software Interrupts (SWIs) Used
*   `SWI 3`: Writes a null-terminated string (passed as a pointer in `R0`) to the emulator's output window.
*   `SWI 2`: Terminates program execution.
